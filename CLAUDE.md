# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

"Binary Brew" ‚Äî a personal blog by Alexandros Koutroulis, built with Astro 5, Tailwind CSS, and MDX. Deployed to GitHub Pages at `https://mistericy.github.io`.

## Commands

```bash
npm run dev          # Dev server at localhost:4321 (drafts visible here)
npm run build        # Production build to ./dist/
npm run preview      # Preview the production build locally

# Content authoring helpers
npm run draft "Post Title"                          # Create a new draft post
npm run generate-cover -- --post src/content/blog/slug.md   # AI cover image (requires OPENAI_API_KEY)
npm run tts-post                                    # Text-to-speech via ElevenLabs (requires API key)
```

There are no tests or linting scripts configured.

## Architecture

### Content System

Blog posts live in `src/content/blog/*.md` (Markdown or MDX). The schema is defined in `src/content.config.ts` using Astro Content Collections. Required frontmatter: `title`, `description`, `pubDate`. Important behavior: posts with `draft: true` are filtered out in production builds but visible during development (`npm run dev`).

Frontmatter fields:
- `heroImage` ‚Äî path relative to the post file (e.g. `../../assets/slug-cover.webp`)
- `tags`, `keywords`, `category`, `excerpt` ‚Äî used for SEO and display
- `updatedDate` ‚Äî used by the sitemap serializer in `astro.config.mjs` to set `<lastmod>`

### SEO / Head Components

Two-layer head system in `src/components/`:
- `BaseHead.astro` ‚Äî canonical URL, Open Graph basics, Twitter Card
- `SEOHead.astro` ‚Äî Schema.org JSON-LD (BlogPosting), article metadata, breadcrumbs

Both are composed in `src/layouts/BlogPost.astro`, which is the sole layout for all blog posts. The layout also injects TOC (from Astro's `headings`), reading time (from `src/lib/readingTime.ts`), author bio, and social sharing links.

### Routing

- `/` ‚Üí `src/pages/index.astro`
- `/blog/` ‚Üí `src/pages/blog/index.astro`
- `/blog/[slug]` ‚Üí `src/pages/blog/[...slug].astro` ‚Äî uses `getStaticPaths()` with `getCollection('blog')`
- `/rss.xml` ‚Üí `src/pages/rss.xml.js`
- Sitemap auto-generated by `@astrojs/sitemap` with custom `serialize` in `astro.config.mjs`

### Global Constants

`src/consts.ts` holds `SITE_TITLE`, `SITE_URL`, `SITE_AUTHOR` and social handles. Import from here rather than hardcoding site metadata.

### AI-Assisted Authoring Scripts

`scripts/generate-cover.ts` ‚Äî calls OpenAI (`gpt-4o-mini` for theme extraction, `gpt-image-1.5` for generation) to create a `*-cover.webp` in `src/assets/` and automatically updates the post's `heroImage` frontmatter.

### Agent Specs

`.agents/` contains prompt specs for two Claude agents:
- `image-creation.md` ‚Äî Editorial Visual Prompt Architect (for image generation prompts)
- `seo-optimizer.md` ‚Äî SEO Optimizer (analyzes posts and proposes frontmatter/content improvements)

These define how agents should behave when working on posts in this repo.
